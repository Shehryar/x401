Open Protocol

An open standard for internet-native agent authorization

One token that lets any AI agent prove what it's authorized to do, who authorized it, and when that authorization expires. Verifiable on-chain and off-chain. No vaults, no per-service keys, no human in the loop.

Read the White Paper Design Doc

Verify agent authorization in one line

// npm install @x401/verify

import { verifyACT } from '@x401/verify';

app.get('/api/data', verifyACT({ require: 'https://api.example.com/data:read' }), handler);

Self-contained

An Agent Capability Token carries everything a verifier needs: who authorized the agent, what it can do, quantitative limits, expiry, and the full delegation chain. Verification is a local cryptographic operation.

On-chain native

Smart contracts verify ACTs natively in the VM. ~5,000 compute units on Solana, ~50K gas on EVM. No oracles, no gateways. Combined with x402, an agent proves authorization and pays in a single HTTP roundtrip.

Delegation chains

Human to orchestrator to worker to sub-worker. Each hop carries cryptographic proof of delegation with provable attenuation. The final agent in the chain traces back to the human's original scoped authorization.

We need a new way to authorize agents

The old way of managing credentials was barely working for a human world, let alone an agentic future.

How it works

01

Agent Request

An AI agent requests access to an API, on-chain resource, or any service endpoint.

02

Authorization Required

If no valid ACT is attached, the server responds with HTTP 401 specifying the required capabilities.

03

Agent Attaches ACT

The agent includes its Agent Capability Token in the Authorization: ACT header.

04

Service Verifies

The service verifies the ACT locally (signature chain, capabilities, limits) and serves the request. No round-trip to any external service.
